# 유튜브 트렌드 분석 플랫폼 FAQ

## 플랫폼 소개

### 이 플랫폼은 무엇인가요?

한국 유튜브 트렌딩 데이터를 실시간으로 수집하고 분석하는 AI 기반 분석 플랫폼입니다. 크리에이터, 마케터, 연구자가 유튜브 트렌드를 이해하고 전략적 의사결정을 내리는 데 활용할 수 있습니다.

### 주요 기능

1. **트렌딩 순위 조회**: 실시간 트렌딩 TOP 50 모니터링
2. **카테고리 분석**: 카테고리별 트렌딩 점유율 및 성과 비교
3. **채널 분석**: 채널별 성과 지표 및 순위 추적
4. **콘텐츠 유형 비교**: 쇼츠 vs 일반 영상 성과 분석
5. **성장률 분석**: 급상승 콘텐츠 탐지 및 패턴 분석
6. **AI 인사이트**: 자연어 질문으로 데이터 분석

## 데이터 관련

### 데이터는 얼마나 자주 업데이트되나요?

| 데이터 유형 | 업데이트 주기 |
|:---|:---|
| 트렌딩 순위 | 1시간 |
| 영상 통계 (조회수, 좋아요) | 1시간 |
| 채널 통계 (구독자) | 6시간 |
| 성장률 계산 | 실시간 비교 |

### 어떤 데이터를 수집하나요?

**수집 데이터:**
- 트렌딩 영상 메타데이터 (제목, 설명, 태그)
- 성과 지표 (조회수, 좋아요, 댓글 수)
- 채널 정보 (구독자, 총 조회수)
- 콘텐츠 분류 (카테고리, 쇼츠 여부)
- 시계열 스냅샷 (순위 변동 추적)

**수집하지 않는 데이터:**
- 개인 시청 기록
- 비공개 영상 정보
- 채널 수익 데이터
- 시청자 개인정보

### 데이터 정확도는 어떻게 보장하나요?

- YouTube Data API v3 공식 데이터 사용
- 1시간 단위 스냅샷으로 변동 추적
- 이상치 탐지 및 필터링 적용
- 데이터 무결성 검증 프로세스

## 사용 방법

### 어떻게 질문하나요?

AI 챗봇에 자연어로 질문하면 됩니다.

**질문 예시:**
- "현재 게임 카테고리 트렌딩 TOP 10 보여줘"
- "쇼츠랑 일반 영상 비교해줘"
- "가장 성장률 높은 채널은?"
- "오늘 새로 트렌딩에 진입한 영상은?"

### 어떤 유형의 질문에 답할 수 있나요?

**가능한 질문:**
| 유형 | 예시 |
|:---|:---|
| 순위 조회 | "TOP 10", "몇 위야?" |
| 비교 분석 | "A vs B 비교" |
| 필터링 | "게임 카테고리만", "쇼츠만" |
| 통계 | "평균 조회수", "참여율" |
| 트렌드 | "요즘 뜨는 카테고리" |
| 개념 설명 | "참여율이 뭐야?", "알고리즘 설명" |

**불가능한 질문:**
- 미래 예측 ("내일 뭐가 뜰까?")
- 개별 URL 분석 ("이 영상 분석해줘")
- 수익 정보 ("이 채널 얼마 벌어?")

### 차트/테이블이 안 보여요

차트와 테이블은 분석 가능한 충분한 데이터가 있을 때만 표시됩니다.
- **데이터 충분**: 차트 + 텍스트 요약
- **데이터 부족**: 텍스트 설명만

## 분석 팁

### 더 정확한 분석을 받으려면?

1. **구체적으로 질문하세요**
   - ❌ "순위 보여줘"
   - ✅ "게임 카테고리 조회수 기준 TOP 5"

2. **숫자를 명시하세요**
   - ❌ "많이 보여줘"
   - ✅ "10개 보여줘"

3. **필터를 활용하세요**
   - "엔터테인먼트 카테고리에서 쇼츠만"
   - "구독자 100만 이상 채널"

4. **비교를 요청하세요**
   - "게임 vs 엔터테인먼트 참여율 비교"
   - "오늘 vs 어제 트렌딩 변화"

### 자주 쓰는 분석 패턴

| 목적 | 질문 예시 |
|:---|:---|
| 트렌드 파악 | "요즘 가장 핫한 카테고리는?" |
| 경쟁 분석 | "게임 카테고리 TOP 채널들" |
| 콘텐츠 전략 | "쇼츠 vs 일반, 뭐가 더 효과적?" |
| 성장 기회 | "빠르게 성장하는 채널 특징" |
| 벤치마킹 | "트렌딩 1위 영상 특징" |

## 기술 관련

### 어떤 기술을 사용하나요?

- **데이터 파이프라인**: Apache Airflow, Python
- **데이터베이스**: Supabase (PostgreSQL)
- **AI 모델**: Ollama (로컬 LLM)
- **벡터 검색**: pgvector
- **프론트엔드**: Next.js, TypeScript
- **백엔드**: FastAPI

### API를 직접 사용할 수 있나요?

현재 공개 API는 제공되지 않습니다. 추후 개발자용 API 제공 계획이 있습니다.
